java基础——mac下配置JDK环境变量

By 威威小威

发表于 2015-05-15

以下是针对单独用户配置JDK变量的方法，亲测有效。

#下载jdk7的mac版
#官网下载地址http://www.oracle.com/technetwork/java/javase/downloads/
#下载类似jdk-7u67-macosx-x64.dmg，版本号可能会更新
#安装好之后，进入以下路径查看
cd /Library/Java/JavaVirtualMachines/
#再查看你自己安装的版本
ls 
#进入目录(有多个版本，则手动输入期望的版本；只有一个版本，则输入cd空格后可以直接按tab键自动补全)
cd jdk1.7.0_67.jdk/Contents/Home

#复制jdk home的路径，在当前目录输入pwd
pwd
#把显示出来的路径先复制起来

#进入用户主目录，然后看一下有没有.bash_profile文件
cd ~
ls -all
#编辑.bash_profile文件(没有则新建，命令touch .bash_profile)
vim .bash_profile
#英文模式下，按一下i键进入编辑模式，输入以下内容，路径部分自己粘贴自己的
export JAVA_HOME=/Library/Java/JavaVirtualMachines/jdk1.7.0_67.jdk/Contents/Home
#输完后，按esc，再按冒号(shift+;)，输入wq保存即可

#重启一下终端，输入
java -version
#看到有显示就完成了

面试经历--蘑菇街

By 威威小威

发表于 2015-05-06

今天上午9点去参加了蘑菇街的面试，去面试前，已向同学了解到，蘑菇街校招实习生面试一般会三面一次性进行，两次技术面，一次HR面，如果某一面没过，就会直接让你回来了。我在技术一面后就让回来等通知，应该是被刷了，就当是一次经验积累吧。
刚到蘑菇街公司，在前台会让你填一张个人简要信息的表格，过会面试官过来把这张表格和你在网上提交的简历一起拿去，去11楼一个小房间开始面试。
面试的内容主要以项目为主，从项目展开，问相关的知识点，问得很深，他会抓住一个点，一直不断地问，直到你答不出来为止。
我做过一个家装O2O的网站，自我介绍后，就是让我介绍一下这个项目。介绍完了根据项目问的主要问题：

  1. 采用了DapperPHP框架，请说明采用这个框架的原因。
  2. 给这个框架增加了Lib层，请说明原因
  3. 团队人员组成，每个人分别是做什么的（这个问题面试官主要还是想了解你在这个项目中做了什么，发挥了什么作用）
  4. 当我说后台的数据库设计是我做的，他会问为什么要那样设计，要求当场画ER图；还问订单部分的数据库设计，同样要求是当场画ER图
  5. 中间提到数据库优化，要求介绍一下具体做了哪些优化，为什么要做那些优化，优化后实际的效果是什么，还会要求当场写SQL语句。
  6. 问我在数据库方面看了那些书。（平时书看得少就惨了，我说只看了mysql必知必会，网络上搜索的知识点比较多）

  7. 问在程序开发中最不喜欢的事是什么？（这个问题当时答得太nc了，被问了一个多小时，这个时候已经完全不知道自己在说什么了，居然说不喜欢程序调试时很烦，有时逻辑是对的，但上板调试又出错。答完了之后直接就例行公事问我有什么想问他的，就让我回去等通知了）

    总体来说，面试是以项目展开，知识点会问得很细，很深，所以准备这种面试必须将自己做的项目各个方面研究透，不能被问倒。另外，像问程序开发中最不喜欢的事，千万不能老老实实回答说不喜欢繁琐了，或许可以说沟通不充分，信息不透明，比如我们实验室（实验室的项目是秘密级的）

线性结构的基本特征

  1. 集合中必存在唯一的一个“第一元素”；
  2. 集合中必存在唯一的一个“最后元素”；
  3. 除最后一个元素，均有唯一的后继
  4. 除第一个元素之外，均有唯一的前驱

2.1 线性表的类型定义

  1. 对线性表操作前，一般需要对线性表先进行一个排序操作，排序的时间复杂度最低为nlogn。然后再是对有序表进行操作。

2.2 线性表的顺序表示和实现

  1. 顺序映像指用一组地址连续的存储单元来依次存储线性表中的数据元素。
  2. 插入顺序表元素和删除顺序表元素时间复杂度O(n);
  3. 顺序表的优点：可以对表的元素直接进行读取；缺点：当进行插入或者删除操作时，需要移动之后的所有元素。

2.3 线性表的链式表示和实现

  1. 链式映像用一组地址任意的存储单元来存储数据元素。
    元素+指针=结点；链表：“结点的序列”

数据结构（C语言版）学习总结1--绪论

By 威威小威

发表于 2015-05-05

1.1 什么是数据结构

数据结构描述现实世界实体的数学模型（非数值计算）及其上的操作在计算机中的表示和实现。

1.2 基本概念和术语

  1. 数据：所有能被输入到计算中，且被计算机处理的符号的集合。计算机操作的对象的总称。
  2. 数据元素：数据的基本单位。
  3. 数据项：数据结构中讨论的最小单位，数据元素是数据项的集合。
  4. 数据结构：相互之间存在一种或多种特定关系的数据元素的集合。
    关系：1）线性结构 2）树形结构 3）图状结构 4）集合结构
    数据结构的形式化定义：Data_Structure = (D,S)
    D是数据元素的有限集，S是D上关系的有限集。
  5. 数据结构在计算机中的表示（映像）：
    1）数据元素的映像表示：二进制位串
    2）数据关系的映像表示：顺序映像（以存储位置的相邻表示后继关系），链式映像（以附加信息—指针表示后继关系）
  6. 数据类型
    数据类型是一个值的集合和定义在这个值集上的一组操作的总称
  7. 抽象数据类型（ADT）
    指一个数学模型以及定义在该模型上的一组操作
    两个重要特征：
    1）数据抽象
    2）数据封装
    描述方法：（D,S,P）P为对D的操作集

1.3 算法和算法分析

  1. 算法
    算法是为了解决某类问题而规定的一个有限长的操作序列。
    5个重要特征：1）有穷性 2）确定性 3）可行性 4）输入 5）输出
  2. 算法设计的原则
    1）正确性 2）可读性 3）健壮性：处理出错的方法不应该是中断程序的进行，而应是返回一个表示错误或错误性质的值，以便在更高的抽象层次上进行处理 4）效率与低存储量需求
  3. 算法效率的衡量方法和准则
    时间复杂度 T(n)=O(f(n))
    空间复杂度 S(n)=O(f(n))
  4. 如何估算时间复杂度
    算法 = 数据结构 + 原操作（固有数据类型的操作）
    算法的执行时间 与 原操作的执行次数 成正比，因此可以选取一种基本操作的原操作，以基本操作在算法中的重复执行的次数，作为算法运行时间的衡量准则。
    如选择排序，时间复杂度O(n^2)
    冒泡排序，最好的情况下为0，在最坏情况下时间复杂度O(n^2)

leetcode(1)-Two Sum

By 威威小威

发表于 2015-04-27

题目

Given an array of integers, find two numbers such that they add up to a specific target number.

The function twoSum should return indices of the two numbers such that they add up to the target, where index1 must be less than index2. Please note that your returned answers (both index1 and index2) are not zero-based.

You may assume that each input would have exactly one solution.

Input: numbers={2, 7, 11, 15}, target=9
Output: index1=1, index2=2

分析

1、暴力求解，直接做两层循环找出两个数所在的位置，但时间复杂度为O(n^2)，会超时
2、先进行一个快排，两个指针分别指向首(i)尾(j)，若和大于要求值则 j—，否则i++

代码

#include <stdio.h>
/*
 1、暴力求解，时间复杂度为O(n^2),会超时: Time Limit Exceeded
 */
int* twoSum1(int* nums, int numsSize, int target) {
    int *a = (int *)malloc(2*sizeof(int));
    int *index=(int *)malloc(numsSize*sizeof(int));
    int i=0;
    int j=0;
    for(i=0;i<numsSize;i++){
        index[i] = i+1;
    }
    for(i=0;i<numsSize;i++){
        for(j=i+1;j<numsSize;j++){
            if(nums[i]+nums[j] == target){
                a[0] = index[i];
                a[1] = index[j];
                break;
            }
        }
    }
    return a;
}
/*
 2、先进行一个快排，两个指针分别指向首(i)尾(j)，若和大于要求值则 j--，否则i++
 */
void swap(int *x, int *y)
{
    int temp;
    temp = *x;
    *x = *y;
    *y = temp;
    return;

}

//快速排序函数，另外一个数组是为了交换时下标也跟着交换
void sort(int arr[],int ind[],int left,int right)
{
    if(left>right)
        return;
    int i=left;
    int j=right;
    int mid=(left+right)/2;
    swap(&arr[mid],&arr[left]);
    swap(&ind[mid],&ind[left]);
    int temp=arr[left];
    while(i!=j)
    {
        while(arr[j]>=temp && i<j)
            j--;
        while(arr[i]<=temp && i<j)
            i++;

        if(i<j)
        {
            swap(&arr[i],&arr[j]);
            swap(&ind[i],&ind[j]);
        }
    }
    swap(&arr[i],&arr[left]);
    swap(&ind[i],&ind[left]);

    sort(arr,ind,left,i-1);
    sort(arr,ind,i+1,right);

}

int *twoSum2(int* nums, int numsSize, int target) {
    int *a=(int *)malloc(2*sizeof(int));
    int *index=(int *)malloc(numsSize*sizeof(int));
    for(int i=0;i<numsSize;i++)
        index[i]=i+1;
    a[0]=a[1]=-1;
    sort(nums,index,0,numsSize-1);
    int i=0;
    int j=numsSize-1;
    while (i!=j) {
        if(nums[i]+nums[j]==target)
        {
            if(index[i]>index[j])
            {
                a[0]=index[j];
                a[1]=index[i];
            }
            else
            {
                a[0]=index[i];
                a[1]=index[j];
            }
            break;
        }
        else if(nums[i]+nums[j]<target)
            i++;
        else
            j--;
    }
    return a;
}

int main(int argc, const char * argv[]) {
    // insert code here...
    //printf("Hello, World!\n");
    int nums[6] = {2,3,6,1,4,7};
    int numsSize = 6;
    int target = 13;
    int *indexCur;
    indexCur = twoSum2(nums, numsSize, target);
    printf("%d\n%d\n",indexCur[0],indexCur[1]);
    return 0;
}

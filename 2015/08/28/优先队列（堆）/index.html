2. 优先队列的典型实现——堆

用二叉堆来实现优先队列相当普遍，甚至于优先队列和堆这两个概念基本可以等同使用。
二叉堆具有两个性质：

  • 结构性
  • 堆序性
2.1 结构性

二叉堆在结构上时一棵完全二叉树。另外，因为完全二叉树的规律性，可以用数组来表示。
下图为二叉堆的完全二叉树结构，以及对应的数组。

2.2 堆序性

在一个堆中，对于每一个节点X,X的父节点中的关键字都小于（或等于）X中的关键字，根节点除外（它没有父节点）。

3. 堆操作

对堆操作的关键在于保持堆序性

3.1 insert

思路：（1）为将一个元素X插入到堆中，我们在下一个可用位置创建一个空穴,否则该堆将不是完全树。
（2）如果X可以放在空穴而不破坏堆序性，那么插入完成；
（3）否则，我们把空穴的父节点上的元素移入空穴中，这样，空穴就朝着根的方向上冒一步。继续该操作直到X能放入空穴为止。这样的策略称为上滤

图例：

java代码：

public void insert(AnyType x){
    //存二叉堆的数组已满时，需要给数组扩容
    if(currentSize == array.length-1)
        enlargeArray(array.length*2+1);

    //上滤
    int hole = ++currentSize;
    for(;hole>1 && x.compareTo(array[hole/2])<0;hole /=2)
        array[hole] = array[hole/2];
    array[hole] = x;
}

插入的运行时间最大为O(log N)。

3.2 deleteMin

思路：deleteMin的处理方式类似于insert。找出最小元是容易的，困难之处在删除它。
（1）当删除一个最小元时，需要在根节点建立一个空穴。
（2）由于现在堆少了一个元素，因此堆中最后一个元素X必须移到该堆的某个地方。如果X可以放到空穴中，则deleteMin完成；
（3）否则，将空穴的两个儿子中较小者放到空穴中，这样就把空穴向下推了一层。重复该操作，直到X可以放入空穴中。这种策略称为下滤

图例：

java代码：

public AnyType deleteMin(){
    if(isEmpty())
        throe new UnderflowException;

    AnyType minItem = findMin();
    array[1] = array[currentSize--];
    percolateDown(1);

    return minItem;
}

//下滤操作的具体实现
private void percolateDown(int hole){
    int child;
    //tmp存放最后一个节点的关键字X
    AnyType tmp = array[hole];

    //找到可以存放X的空穴
    for(;hole*2<=currentSize;hole=child){
        child = hole*2;
        if(child != currentSize && array[child+1].compareTo(array[child])<0) //如果右儿子的值小于左儿子的值
            child++;
        if(array[child].compareTo(tmp)<0)
            array[hole] = array[child]; //用儿子中较小者来填充空穴
        else
            break;
    }

    array[hole] = tmp;

}

deleteMi的运行时间最大为O(log N)。

4. d-堆

我们一般所说的堆是二叉堆，即2-堆，这就不难理解d-堆了。
d-堆insert操作的运行时间为O(log d N)；deleteMin操作的运行时间O(d*log d N)。
d-堆类似于B树，当堆太大而不能完全装入内存的时候，d-堆也很有用。

5. 左式堆

在理解左式堆前，需要理解一个概念：零路径长。

我们把任一节点的零路径长（null path length）npl(x)定义为从X到一个不具有两个儿子的节点的最短路径的长，具有0个或者1个儿子的节点的npl为0，而npl(null)=-1.

零节点图例：

下面来定义左式堆：

对于堆中的每一个节点X,左儿子的零路径长大于等于右儿子的零路径长，这样的堆称为左式堆。

那么，为什么需要左式堆呢？答案是合并操作（merge），二叉堆难以将两个堆合并，左式堆的特殊性便于进行合并。

merge操作

思路：（1）如果两个堆中有一个为空的，那么返回另一个堆；
（2）否则，合并两个堆。首先将具有大的根值的堆与具有小的根植的堆的右子树进行合并，得到一个新的左式堆。
（3）接着，将剩下的左子树接到新的左式堆上。
（4）判断最后得到的堆的左子树和右子树的零路径长，如果不符合左式堆的定义，则交换左右子树。

5. 斜堆

斜堆跟左式堆的关系就相当于伸展树跟AVL树的关系，对斜堆进行M次连续操作，总的最坏情形运行时间为O(M log N),摊还开销为O(log N).

6.二项队列

二项队列不同于左式堆和二叉堆等优先队列的实现之处在于，一个二项队列不是一棵堆序的树，而是堆序树的集合，即森林。堆序树中的每棵树都是由约束形式的，叫做二项树。每一个高度上至多存在一棵二项树。高度为0的二项树是一颗单节点树，高度为k的二项树Bk通过将一棵二项树Bk-1附接到另一颗二项树Bk-1的根上而构成。

二项队列的性质：

  • 高度为k的二项树恰好有2^k个节点；
  • 深度为d处（这里的深度是以根的深度为0计算的）的节点数为（k d）。

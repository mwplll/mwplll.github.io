排序（四）-希尔排序

希尔排序的名称来源它的发明者Donald Shell,该算法是冲破二次时间屏障的第一批算法之一。

1. 算法思路

希尔排序的基本原理：
（1）首先构造一个增量序列：h1,h2,…,hk,其中h1=1;
（2）使用hk进行一趟排序，排序的目标是保证相隔hk的元素被排序，即a[i]<=a[i+k]。实质是对hk个子数组进行一次插入排序
（3）接着依次使用h(k-1),…,h2,h1进行排序，当进行到增量为h1=1时，得到的即为有序序列。

希尔排序的正确性首先依赖于一个性质：一个hk排序的文件（然后将是h(k-1)排序的）保持它的hk排序性

2. java代码

源码：https://github.com/mwplll/algorithms

package sort;

public class ShellSort {
    /**
     * 希尔排序
     * 增量序列采用希尔建议的序列：ht=N/2;hk=h(k+1)/2
     * @param a
     */
    public static <AnyType extends Comparable<? super AnyType>>
    void shellSort(AnyType[] a){
        int j;

        for(int gap = a.length/2;gap>0;gap /= 2){
            for(int i = gap;i<a.length;i++){
                AnyType tmp = a[i];
                for(j = i;j >= gap && tmp.compareTo(a[j-gap])<0;j -= gap)
                    a[j] = a[j-gap];
                a[j] = tmp;
            }
        }
    }

    public static void main(String[] args) {
        Integer[] a = {38,65,97,76,13,27,49};
        shellSort(a);
        for(int i=0;i<a.length;i++){
            System.out.print(a[i]+" ");
        }
    }
}

3. 算法分析

两个定理记住：

  • 使用希尔增量时希尔排序的最坏情形运行时间为O(n^2);
  • 使用Hibbard增量的希尔排序的最坏情形运行时间为O(n^(3/2))

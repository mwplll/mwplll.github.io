1. 什么是二叉查找树，有什么特点

对于树中的每一个结点X,它的左子树中的所有项的值都小于X中的项，它的右子树中的所有项的值都大于X中的项。这样的树称为二叉查找树

二叉查找树的性质：

  • 其深度的平均值是O（logN）。
  • 查找、插入、删除平均时间O(logN)。
  • 对二叉查找树执行一次中序遍历，得到的是排过顺序的元素，算法复杂度为O(N log N);

2. 二叉查找树的操作

2.1 contains方法

java代码：

private boolean contains(AnyType x,BinaryNode<AnyType> t){
    //首先对是否为空树进行测试，避免产生NullPointerException异常
    if(t==null)
        return false;

    int compareResult = x.compareTo(t.element);

    if(compareResult<0)
        return contains(x,t.left);
    else of(compareResult>0)
        return contains(x,t.right);
    else
        return true;  //匹配
}
2.2 findMin和findMax方法

java代码：

/**
* 用递归形式编写的findMin方法
*/
private BinaryNode<AnyType> findMin(BinaerNode<AnyType> t){
    if(t==null)
        return null;
    else if(t.left==null)
        return t;
    //从左子树寻找最小值
    return findmin(t.left);
}

/**
* 用while循环编写的findMax方法
*/
private BinaryNode<AnyType> findMax(BinaerNode<AnyType> t){
    if(t==null)
        return null;
    while(t.right!=null)
        //从右子树寻找最大值
        t = t.right;

    return t;
}
2.3 insert方法

java代码：

private BinaryNode<AnyType> insert(AnyType x, BinaryNode<AnyType> t) { 
     //递归结束条件，在子树为空的地方插入该结点
    if (t == null) {  
        return new BinaryNode<AnyType>(x, null, null);  
    }  

    int compareResult = x.compareTo(t.element); 

    if (compareResult > 0) {  
        t.right = insert(x, t.right);  
    } else if (compareResult < 0) {  
        t.left = insert(x, t.left);  
    } else {  
        ;  //重复，不用做任何事
    }  
    return t;  
}
2.4 remove方法

删除操作考虑三种情况

  1. 如果节点是一片树叶，则可以立即被删除
  2. 如果节点有一个儿子，则可以在该节点的父节点调整自己的链以绕过该节点后删除
  3. 如果节点有两个儿子，一般的删除策略是找到右子树中的最小值，来代替被删除节点的数据，并递归地删除那个节点（涉及两次删除）

java代码：

private BinaryNode<AnyType> remove(AnyType x, BinaryNode<AnyType> t) {  
    if (t == null) {  
        return null;  
    } 

    int compareResult = x.compareTo(t.element);  

    if (compareResult > 0) {  
        t.right = remove(x, t.right);  
    } else if (compareResult < 0) {  
        t.left = remove(x, t.left);  
    } else if (t.left != null && t.right != null) {  //有两个儿子的情况
        t.element = findMin(t.right).element;  
        t.right = remove(t.element, t.right);  
    } else if (t.left != null) {  //只有一个儿子或者为叶子节点的情况
        t = t.left;  
    } else {  
        t = t.right;  
    }  
    return t;  
}

如果删除的次数不多，通常使用的策略是懒惰删除：当一个元素被删除时，它仍然留在树中，而只是被标记为删除。

对于有两个儿子的节点的删除策略，可以用右子树的最小值或者左子树的最大值来代替该节点，如果全部用右子树的最小值来代替该节点，则经过多次操作后，会明显破坏二叉树的平衡。
